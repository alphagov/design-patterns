Design guide examples

These examples use the GOV.UK boilerplate, which provides basic styles for typography and grid layout. It shows how to make use of the Sass variables and mixins provided by the GOV.UK frontend toolkit.

To keep things tidy, typography styles are applied using the .gov-uk class on the <main> content area.

  • Form styles and buttons

Using Sass

These examples assume you are familiar with Sass and that you will use Sass to generate your stylesheets. In these examples, base.scss is the file that will be compiled with Sass.

Take a look at the base.scss file to see how these examples are structured.

The GOV.UK boilerplate

Dependancies

The GOV.UK boilerplate main.scss has a dependancy on the GOV.UK frontend toolkit.

You'll need the GOV.UK frontend toolkit to use the GOV.UK boilerplate, since it uses the mixins and variables the toolkit defines.

Best practice

The main.scss file is structured in a mobile-and-up fashion. Mobile styles are defined first, then the media mixin is used to override the default mobile styles with tablet or desktop styles.

Variables in Sass make it easy to see where the same numbers are repeated. The boilerplate uses a $gutter variable a lot! This will give you 30px and $gutter-half at 15px.
