Design guide examples

These examples show how the GOV.UK boilerplate is used to create basic styles for typography and grid layout.

To keep things tidy, typography styles are applied using the .gov-uk class on the <main> content area. The .prose class is used to wrap blocks of text, for an optimal line length.

Using Sass

These examples assume you are familiar with Sass and that you will use Sass to generate your stylesheets. In these examples, base.scss is the file that will be compiled with Sass.

Take a look at the base.scss file to see how these examples are structured.

The GOV.UK boilerplate

Dependancies

The GOV.UK boilerplate main.scss file has a dependancy on the GOV.UK frontend toolkit.

You'll need the GOV.UK frontend toolkit to use the GOV.UK boilerplate, since it uses the mixins and variables the toolkit defines.

Best practice

The main.scss file is structured in a mobile-and-up fashion. Mobile styles are defined first, then the media mixin is used to override the default mobile styles with tablet or desktop styles.

Variables in Sass make it easy to see where the same values are repeated. The boilerplate makes use of a $gutter variable. This is 30px and there's also a $gutter-half at 15px. Take a look at the grid layout to see these in use.
